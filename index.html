RemoteDebug

Initiative to unify remote debugging across browsers.

The initiative.

RemoteDebug is an initiative to bring a common remote debugging protocol to today's browsers. It isn't a new phenomenon. It has been around for years, but today each browser got their own, so we got:

  • Chrome Remote Debugging Protocol.
  • Opera Scope Transport Protocol (STP).
  • Mozilla Remote Debugging Protocol.
  • WebKit Remote Debugging Protocol (WIP).

Each are incompatible with each other, which means existing DevTools can't debug other browsers, and tooling implementors need to spent a lot of time adding support for each protocol (and this means nobody does it).

The vision for this initiative is to enable in-browser DevTools and editors to communicate and remote multiple browsers. Imagine if it was possible to use Chrome DevTools to debug Firefox, or use Firefox DevTools to debug Chrome. What if you could debug your app in IE and Chrome directly from Sublime? When running your beautiful PhoneGap-app inside an embedded WebView, you should be able to use your favorite debugging tool, and not be forced you to use Weinre.

We are building for the open web. Our tooling and protocols should be open too.

The plan.

So how are we going to make this happen? Instead of starting from scratch, we propose to use the existing Chrome Remote debugging protocol.

Why, you might ask? The Chrome protocol is [well-documented](link), and is an open protocol. The protocol is used by Chromium-based browsers, which means we, out of the box, got Chrome and Opera supported. In addition we also got a Webkit-protocol-to Chrome proxy implemented, so by having three browsers supported, we think we are off to a great start.

Get involved.

There's a lot of work ahead, so we need your help. You can help by either spreading the word, by contributing to one of the existing projects, or maybe even start you own?

If you do your own thing, let us know, so we can add it to the project list.

Implementations and tools.

  • ios-webkit-debug-proxy

    A DevTools proxy (WebKit Remote Debugging Protocol) for iOS devices (Safari Remote Web Inspector).

  • node-inspector

    Node debugger based on Blink Developer Tools

  • node-chrome-remote-debug

    Node client for the Chrome Remote debugging protocol.
